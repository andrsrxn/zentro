# Commit Message Guidelines

- Always use the Conventional Commits format (e.g., feat: add login).
- The subject line must not exceed 50 characters.
- Use the imperative mood (e.g., "Add feature" instead of "Added feature").
- Base the message strictly on staged changes (`git diff --staged`).
- If the change is complex, include a body explaining the "why" behind the change.
